Intro to Recurrent Neural Networks

Recurrent Neural Networks or RNNs are especially powerful in handling sequential data. Sequential data can come in the form of audio or text and RNNs allow us to do many tasks in natural language processing such as trigger word detection which is simply how your Google Home or your Alexa detects when you're talking to them.

Recurrent Neurons

Previously, we mostly looked at feedforward neural networks where the activations flow in one direction from an input layer to an output layer. A recurrent neural network adds an extra step which is outputting the previous layer's answer as the input of the next layer's.

Notice how the each layer receives the previous layer's outputs.

Each cell is able to remember the previous cell's output which allows for sequential memory which allows the model to be accurate based on patterns that it has learned before.

What can we do with this?

  1. Many to One Architecture: We could have the entire RNN run and the ignore all of its outputs except for the last one. This means our input sequence is many different words but our output is just one "score." This is powerful in sentiment classification where we could have someone say "The Food is amazing!" and the classifier would output 1 for Good.
  1. Many to Many Architecture: An example of this where the input sequence length matches the output sequence length would be stock prices. For example, let's say we have data over the last N days, and we want to output the prices shifted by one day into the future. This would then require each previous day to output one corresponding to the next day.
  1. One to Many Architecture: An example of this is if we had an image and who want to caption it. We would then receive one input as the image vector and we would have to output a sequence as the output
  1. Encoder-Decoder Architecture: This is great for language models where the input length might not match the output. We would then use an encoder network would convert the sequence from one language into a single vector representation. Then, the decoder network would output the representation into another language.

Training RNNs

  • In order to train an RNN, the trick is to do regular backpropagation known as backpropagation through time
  • We feed the output into a cost function which calculates how far off our model is from the actual answer
  • We then update the parameters based on the gradients computed during backpropagation

Conclusion

You now know how a Recurrent Neural Networks function, what you can do with it, as well as how to train it. Be prepared to learn how to code it next!
