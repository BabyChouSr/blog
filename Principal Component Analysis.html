Principal Component Analysis

Posted by Christopher Chou on April 16, 2020

Principal Component Analysis

The motivation behind using Principal Component Analysis is to solve the problem referred to as the curse of dimensionality. This is essentially the problem where there are thousands or even millions of features for each training instance which makes training extremely slow.

Dimensionality Reduction

In most real-word problems, the training instances are not spread out uniformly across all dimensions but instead highly correlated. Thus, we can often project a high dimension subspace to a lower-dimensional subspace.

Through Principal Component Analysis, we could project the data from a 2-Dimensional Subspace to a 1- Dimensional subspace along the "PCA 1st Dimension."

PCA Algorithm

  1. Standardize the data
  1. Compute the eigenvectors and the eigenvalues
  1. Compute the Covariance Matrix of the dataset
  1. Sort the eigenvectors and eigenvalues by decreasing value
  1. Pick the first k number of eigenvectors
  1. Use these eigenvectors to transform the samples onto the new subspace

Theory Behind PCA

  1. The Covariance matrix tells us the relationship between the variables.
  1. Eigenvectors give us the direction of the data while the Eigenvalues represent the magnitude or importance. Bigger values indicate the more important directions.
  1. The higher the eigenvalue, the higher the variance which indicates higher importance that we want to detect.

When to use PCA

  1. Reduce the number of variables taken into consideration
  1. To make variables independent of each other
