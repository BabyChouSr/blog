K-Nearest Neighbors Basics

Posted by Christopher Chou on April 16, 2020

K-Nearest Neighbors Algorithm

The K-Nearest Neighbors Algorithm is a powerful Machine Learning classification algorithm that works on supervised data.

Supervised Machine Learning

Supervised data is essentially data that we have both the input and the output of. Pretend we are told to look at a picture of a dog and we have to label it.

After looking at a picture of this, we would all scream dog! However, what if the animal was some mythical creature that we didn't know the name of? Because we don't know the name of the creature, we can't label it which is why it's unsupervised machine learning.

Now that we get a gist of the difference between the two, let's jump into classification.

Classification

A classification algorithm has discrete values as its output. It comes out with either a 1 or 0 telling whether or not it is part of a class.

If we were to have for example, a question which outputs "1" if the person likes smoking and "0" if the person doesn't. Notice how if we add "person likes smoking" + "person doesn't like smoking," it really doesn't mean anything. These labels of numbers are purely representational and don't have any value.

The KNN Algorithm

Now we are ready to begin with the Algorithm. The entire algorithm is based on the fact that similar points or objects are closer together. In order to find how close objects are, we have to use a distance measurement which is usually the Euclidean distance because it is easy to implement.

As an example, notice how the green dots are closer together and the red dots are close together on the bottom left? We try to use the numerical features to find the dots closest to each other.

The Implementation will go along the lines of this:

  1. Load the Data
  1. Pick the Starting "K" # of Neighbors
  1. For each example point in the data
    1. Calculate the distance between this data and every other data and add it to a list
    1. Sort the list so we get an understanding of the order of distances
  1. Pick the first "K" closest neighbors to that point
  1. Get the labels of the "K" entries (1s or 0s)
  1. Print out the mode of the labels which will tell us the class of the one we're predicting.

Value of K

We have to be able to pick the best value of K when it comes to our classification algorithm. Note a couple important tips

  1. A K value that is too low is not very stable. Imagine if we had a K of 1 or 2. We are basing the new point's class off of just 1 or 2 neighbors which can easily be skewed if that 1 or 2 are different.
  1. A K value that is too high is increasingly less stable because it takes the average over a big population. Imagine if we wanted to classify the likelihood someone votes democrat in California. Would it be right to include the entire nation? No, it would be less likely.

Advantages

  1. Algorithm is easy to implement
  1. No Hyperparameters to Tune

Disadvantages

  1. Computational Speed Scales with Data Volume Making it Slow on large Datasets

Conclusion
