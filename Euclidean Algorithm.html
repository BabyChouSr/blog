Euclidean Algorithm

Today, we will be going over how the Euclidean algorithm works to find the greatest common divisor between two numbers. Mathematically, the Euclidean algorithm takes two numbers aaa﻿ and bbb﻿ and find the remainder of it, rrr﻿. If r=0r = 0r=0﻿, then the algorithm is finished, returning bbb﻿. If not, we find the remainder of bbb﻿ divided by r.r.r.﻿ We repeat this until the remainder is 0 and return the divisor.

Code in C++

int GCD(int a, int b ){
    int r = a %b;
    if (r == 0) return b;
    GCD(b,r);
}

This algorithm is extremely simple. All we have to do is take in two numbers aaa﻿ and bbb﻿ then find the remainder rrr﻿. If r is 0 then we return bbb﻿ since bbb﻿ already is the greatest common divisor.

If not, we run the algorithm again, with a→b and b→r instead.

This creates a chain of divisions like this:

GCD=GCD(a,b)→a%b+r=GCD(b,r)→b%r+r2=GCD(r,r2)→r%r2+r3GCD = GCD(a,b) → a \%b + r \\ \hspace{10mm}= GCD(b,r) → b \% r + r_2 \\ \hspace{10mm} =GCD(r,r_2) → r \% r_2 + r_3 GCD=GCD(a,b)→a%b+r=GCD(b,r)→b%r+r2​=GCD(r,r2​)→r%r2​+r3​﻿

all the way until the remainder = 0.

We can use this algorithm to also find the LCM since the LCM=a⋅bGCDLCM = \frac{a\cdot b}{GCD}LCM=GCDa⋅b​﻿

Let's use this algorithm on some numbers.

Using a = 1470 and b = 1155, we get the following output:

Remainder of 1470 and 1155 is 315
Remainder of 1155 and 315 is 210
Remainder of 315 and 210 is 105
Remainder of 210 and 105 is 0
GCD of 1470 and 1155 is 105
LCM of 1470 and 1155 is 16170

Therefore, the GCD is 105 and the LCM is 16170.
